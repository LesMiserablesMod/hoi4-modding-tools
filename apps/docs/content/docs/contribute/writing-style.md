---
title: Writing Style
description: Style guidance for catalog and documentation contributions.
---

## Be specific

Describe what the tool does, what workflow it supports, and what kind of user should consider it.

## Separate facts from judgment

Prefer verifiable statements:

- Supported platforms
- Repository link
- Release page
- License
- Maintenance status

Avoid unsupported claims such as "best" or "most powerful" unless the page explains the basis for that comparison.

## Mention uncertainty

If you cannot verify the status, license, or compatibility of a tool, mark it as `Unknown` rather than guessing.

## Respect third-party projects

This project links to third-party tools and describes them. Do not copy third-party source code, binaries, images, or documentation unless the license clearly permits it and maintainers agree.
